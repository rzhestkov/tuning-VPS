# VPS Setup Scripts

Автоматическая настройка свежих VPS-серверов с защитой от блокировки. 
Скрипты настраивают SSH, файрвол, Docker и автоматические обновления.

## ⚡ Быстрый старт (Ubuntu 24.04)

 1. Подключитесь к серверу как root
```bash
ssh root@YOUR_SERVER_IP
```

2. Скачайте скрипт
```bash
curl -fsSL https://raw.githubusercontent.com/rzhestkov/tuning-VPS/main/setup_ubuntu_24.04.sh -o setup.sh
```

# 3. Запустите
```bash
sudo bash setup.sh
```

## 🔑 Подготовка перед первым запуском

### 1. Поместите SSH ключ в репозиторий

В папке ssh должен лежать готовый файл authorized_keys

## 2. Настройте переменные в скрипте

Убедитесь, что в `setup_ubuntu_24.04.sh` верно прописаны:

```bash
GITHUB_USER="YOUR_GITHUB_USERNAME"  # Ваш аккаунт на GitHub
REPO_NAME="vps-setup"               # Название этого репозитория
NEW_USER="user1"                    # Имя нового пользователя (можно оставить)
SSH_PORT="2332"                     # Порт для SSH (можно изменить)
```

## 🚀 Детальная инструкция

### Шаг 1: Первое подключение

Подключитесь к свежему VPS через веб-консоль хостинга или SSH как `root` с временным паролем.

### Шаг 2: Запуск скрипта

```bash

# Просмотр (опционально, но рекомендуется)
cat setup.sh | less

# Запуск
chmod +x setup.sh
sudo ./setup.sh
```

### Шаг 3: Контрольная проверка (ВАЖНО!)

**Не закрывайте текущее окно терминала, открытое под root!**

Откройте **новое окно** на вашем компьютере c авторизацией по ssh-ключу на порту 2332 и проверьте подключение:

```bash
ssh -p 2332 user1@SERVER_IP
```

Если подключение работает — вернитесь в первое окно и нажмите `yes` для закрытия старого порта 22.

## 🛡️ Что делает скрипт

| Этап | Описание | Защита от ошибок |
|------|----------|------------------|
| **01. Проверки** | Проверка root, Ubuntu, GitHub | Останавливает если не root или нет связи с GitHub |
| **02. Обновление** | `apt update && upgrade` + mc | Проверка подключения к сети |
| **03. Отключение сервисов** | Отключение ненужных сервисов | snapd, apport, whoopsie, lxd, udisks2 |
| **04. Временная зона + NTP** | Устанавливает Europe/Berlin и синхронизацию | Автоматическая настройка через timedatectl, проверка systemd-timesyncd |
| **05. Hardening ядра** | Настройка `/etc/sysctl.conf` | Защита от SYN-flood, отключение IP forwarding, kernel.dmesg_restrict |
| **06. Ограничения ресурсов** | Настройка `/etc/security/limits.conf` | Защита от fork-bomb, ограничение процессов и файлов |
| **07. Пользователь** | Создаёт `user1`, добавляет в sudo | Проверка существования - пропускает если уже есть |
| **08. SSH ключи** | Качает ключ с GitHub | Бэкап существующих ключей + предупреждение о замене |
| **09. SSH настройка** | Меняет порт на 2332, отключает пароли | Бэкап конфигов, `sshd -t`, не закрывает 22 сразу |
| **10. Firewall** | UFW: разрешает 2332, 80, 443 | Сначала открывает новый порт, потом включает файрвол |
| **11. Закрытие порта 22** | Удаляет правило для стандартного SSH | **Только после вашего подтверждения**, проверяет если уже закрыт |
| **12. Fail2ban** | Защита SSH от брутфорса | 3 попытки, бан на 1 час, настроен на кастомный порт SSH |
| **13. Auditd** | Аудит действий на сервере | Логирование изменений системных файлов, SSH конфигурации |
| **14. Автообновления** | Настраивает `unattended-upgrades` | Только security-обновления, без автоматической перезагрузки |
| **15. Needrestart** | Автоматический перезапуск сервисов | Перезапуск сервисов после обновлений без интерактивного режима |
| **16. Journald** | Настраивает системный журнал | Persistent storage, лимиты 500МБ, сжатие, срок хранения 1 месяц |
| **17. Logrotate** | Настраивает ротацию логов | Ежедневная ротация, 7 копий, сжатие для системных логов |
| **18. MOTD** | Кастомное сообщение при входе | Убрана информация о версии ОС, только безопасное сообщение |
| **19. SSH-audit** | Проверка SSH конфигурации | Анализ безопасности SSH с помощью ssh-audit |
| **20. Диагностика** | Отчёт о настройках + проверки  пакетов| Статусы: порт, SSH, UFW, вход по паролю откл., fail2ban, Python, Docker, Nginx, MySQL, Redis и др. |
| **21. Docker** | Устанавливает Docker CE (опционально) | Пропускает если уже установлен |
| **22. MTProto** | Устанавливает MTProto в Docker (опционально) | Пропускает если уже установлен |

### Проверки при повторном запуске

Скрипт оптимизирован для повторного запуска:
- **Пользователь** — если `user1` уже существует, создание пропускается
- **SSH ключи** — если `authorized_keys` уже есть, создаётся бэкап и выводится предупреждение
- **UFW правила** — проверяет существование правил перед добавлением
- **Порт 22** — проверяет, закрыт ли порт, и не задаёт вопрос если уже закрыт
- **Docker** — пропускает установку если уже есть

## ⚠️ Безопасность и защита от блокировки

### Почему вы не потеряете доступ:

1. **Двойное окно**: Скрипт просит подтвердить закрытие порта 22 только после ручной проверки подключения на новом порту
2. **Бэкапы**: Все конфиги SSH сохраняются с timestamp (`sshd_config.bak.1234567890`)
3. **Проверка синтаксиса**: Перед перезапуском SSH проверяется `sshd -t`
4. **Cloud-init fix**: Специально обрабатывает `50-cloud-init.conf`, который на некоторых хостингах (VDSina, Selectel) сбрасывает настройки SSH

### Если всё же потеряли доступ:

Через веб-консоль хостинга (VNC):

```bash
# Восстановить SSH конфиг
sudo cp /etc/ssh/sshd_config.bak.* /etc/ssh/sshd_config
sudo systemctl restart ssh

# Или отключить файрвол
sudo ufw disable
```

## 🔍 Диагностика после установки

Скрипт выводит отчёт в конце, но вы можете проверить вручную:

```bash
# Проверка SSH
sudo systemctl status ssh
sudo ss -tlnp | grep ssh

# Проверка файрвола
sudo ufw status verbose

# Проверка Docker
docker --version
sudo -u user1 docker ps

# Проверка автоматических обновлений
systemctl list-timers apt-daily-upgrade.timer
cat /var/log/unattended-upgrades/unattended-upgrades.log | tail -5

# Проверка временной зоны и времени
timedatectl
timedatectl status | grep "System clock synchronized"

# Проверка systemd-timesyncd
sudo systemctl status systemd-timesyncd

# Проверка journald (системный журнал)
sudo journalctl --disk-usage
sudo systemctl status systemd-journald

# Проверка logrotate
sudo logrotate -d /etc/logrotate.d/custom-system

# Проверка fail2ban
sudo systemctl status fail2ban
sudo fail2ban-client status sshd

# Проверка предустановленных пакетов (что делает скрипт автоматически)
python3 --version
docker --version
nginx -v
mysql --version
redis-server --version
```

## 🐛 Troubleshooting

### "Настройки SSH сбрасываются после ребута"

**Причина**: Cloud-init перезаписывает конфиг.
**Решение**: Скрипт автоматически отключает `50-cloud-init.conf`, но если проблема осталась:

```bash
sudo rm /etc/ssh/sshd_config.d/50-cloud-init.conf
sudo systemctl restart ssh
```

### "UFW блокирует соединение"

```bash
# Временно отключить
sudo ufw disable

# Или добавить свой IP вручную (замените YOUR_IP)
sudo ufw allow from YOUR_IP to any port 2332
```

### "Docker не доступен пользователю"

```bash
# Перелогиниться или выполнить:
sudo usermod -aG docker $USER
newgrp docker
```

## 💡 Советы по использованию

1. **Сделайте репозиторий приватным**, если не хотите светить своим SSH ключом (хотя публичный ключ и безопасен, это просто хорошая практика)

2. **Для автоматизации** можно создать alias у себя на компьютере:
```bash
# Добавьте в ~/.bashrc или ~/.zshrc
alias setup-vps='curl -fsSL https://raw.githubusercontent.com/USERNAME/REPO/main/setup_ubuntu_24.04.sh | sudo bash'
```

3. **Тестирование**: Перед запуском на дорогом сервере, потратьте $1-2 на тестовый VPS у того же провайдера, чтобы проверить совместимость.
